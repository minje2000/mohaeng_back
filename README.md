# 모행 Backend (모두의 모든 행사)

Cloud Native 기반 통합 이벤트 관리 플랫폼  
**누구나 행사를 주최하고, 누구나 행사에 참여할 수 있는 서비스**

---

## 프로젝트 소개

모행은 지역 기반 행사 탐색과 AI 기능을 활용한 이벤트 관리 플랫폼입니다.  
본 저장소는 Spring Boot 기반 백엔드 서버로,  
행사 관리, 리뷰, 알림, 신고 및 AI 검수 연동 기능을 담당합니다.

---

## 담당 역할

- 리뷰 기능 API 구현 (작성, 수정, 삭제, 조회)
- 관심행사(위시리스트) API 구현
- 알림 시스템 API 설계 및 구현
- 행사 신고 및 관리자 처리 API 개발
- **AI 기반 행사 검수 기능 연동 및 백엔드 로직 설계**

---

## 백엔드 구조

- Controller → Service → Repository 구조 기반 설계
- REST API 설계 및 기능별 엔드포인트 분리
- JPA를 활용한 데이터 접근 계층 구성
- JWT 기반 인증 처리

---

## AI 검수 기능 연동 

- 입력: 행사 제목, 한줄 설명, 상세 설명
- 처리: AI 서버(FastAPI)로 요청 전달
- 출력: 위험 점수 (risk_score)

일정 기준 이상의 점수가 산출되면  
해당 행사를 관리자 검수 목록으로 자동 분기하도록 구현했습니다.

백엔드는 AI 서버와의 API 통신을 통해  
검수 결과를 서비스 로직에 반영하는 역할을 수행합니다.

---

## 트러블 슈팅

### 1. 상태 필드 설계 문제
- 문제: 행사 상태와 검수 상태를 하나의 필드로 관리하면서 데이터 누락 발생  
- 해결: 상태를 행사 상태와 검수 상태로 분리하여 독립적으로 관리  
- 결과: 관리자 조회 안정성 확보 및 데이터 정합성 향상  

---

### 2. Git 협업 충돌 문제
- 문제: Git 사용 미숙으로 브랜치 충돌 반복 발생  
- 해결: 브랜치 전략 및 협업 규칙 정립  
- 결과: 협업 효율 향상 및 코드 관리 안정화
